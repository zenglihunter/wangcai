name: google-rank

description: 用 Google CSE 粗略检查某个域名在指定关键词下的排名，大致知道自己在前几页还是掉队了。

---

Usage:

  • Trigger phrase (user): "查看排名, <目标域名>, <关键词>"
  • Example: "查看排名, router-switch.com, fortinet eol"

Behavior:

1. Run Google CSE queries (lang_en + gl=us by default) paginated in blocks of 10 results.

2. For each page, check results for the specified domain (match on displayLink or link hostname). If found, return the absolute rank (page_index*10 + position_in_page).

3. Stop when domain is found or after 10 pages (top 100) exhausted.

4. Return a concise JSON-like output with: {query, target, found:Boolean, rank:Int|null, url:String|null, checked_pages:Int} and a human summary.

Response format requirement (user preference):

  • If the target domain is found: respond succinctly with only the ranking position and the corresponding page URL, one per line. Example:

排名：13

页面：https://www.router-switch.com/fr/cisco-eos-eol.html

  • If not found within max pages: respond succinctly indicating not found, e.g.:

未在前 100 名内找到目标站点。

Configuration:

  • Uses existing GOOGLE_CSE_API_KEY and GOOGLE_CSE_CX env vars.
  • Default params: --language lang_en --country us --num 10 --max-pages 10

Script:

  • scripts/google-rank/search_rank.py

- Arguments: target_domain, query, --max-pages, --language, --country

- Output: JSON pretty-print

Notes:

  • Stop early if the target domain appears in any page.
  • Matching: compare hostname or displayLink endswith target_domain.
  • Respect rate limits: 1s delay between pages by default.

Examples:

python3 scripts/google-rank/search_rank.py router-switch.com "fortinet eol" --max-pages 10
