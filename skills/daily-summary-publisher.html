Daily Summary Publisher

Use this skill whenever it's time to run the nightly history task: every day at 22:58 Asia/Shanghai (or immediately after if that window was missed). The goal is to publish a single canonical post per day that captures everything we did, replacing any earlier draft for the same date.

1. Collect inputs

  • memory/YYYY-MM-DD.md (today's log)
  • Relevant session transcripts or chat exports
  • Open pull requests / commits pushed today
  • Any blockers or follow-ups due tomorrow

Summarize key bullets before writing: deliverables, decisions, risks, next steps.

2. Draft structure

Use a consistent outline so readers immediately know where to look:

  1. 今日概览 – 1–2 句总结今日最重要的事实
  2. 完成事项 – 列表即可，注明影响或输出链接
  3. 问题与处理 – 风险、阻塞、解决进展
  4. 明日/后续 – 下一步、需要 tiger 确认的点

3. Publish / overwrite the day’s post

  1. cd blog-site
  2. Run python3 scripts/new_post.py
  3. Answer prompts:
    • 日期 → 今天（格式 YYYY-MM-DD）
    • 标题 → “2026-03-07 每日总结” 这类可读标题
    • 一句话摘要 → 今日概览
    • 标签 → 例如 日常,总结
    • 发布时间 → YYYY-MM-DD 22:58（若补发，用真实完成时间）
    • 更新时间 → 如果发布后又修改，填当前时间；否则与发布时间相同
    • 输入正文，按照上面的结构逐段粘贴
  4. new_post.py 会自动：
    • 把同一天的旧文章从 data/posts.json 移除
    • 用最新版本覆盖 posts/<slug>.html
    • 重建首页（调用 scripts/render_home.py）

4. Deploy

  1. git status
  2. git add data/posts.json posts/<slug>.html index.html assets/...
  3. git commit -m "chore: daily summary YYYY-MM-DD"
  4. git push https://x-access-token:<token>@github.com/zenglihunter/wangcai.git main
  5. 等待 GitHub Pages build：python3 scripts/render_home.py 已跑完，但要用 API 或网页确认 build=success，再把链接发给 tiger。

5. Scheduling / reminders

  • Treat this as a history task anchored at 22:58。若环境支持 cron/automation，创建一个每天 22:55 的提醒（或脚本调用）来触发本技能。
  • 在缺少自动调度时：把 TODO 写进 HEARTBEAT.md 或本地提醒，确保 22:58 前后手动执行。
  • 如果当晚无法生成（机器离线等），次日上线即补一篇，发布时间 仍写前一日 22:58，更新时间 写补发当刻。

按照以上步骤，任何时候重新运行都能得到“同一天只有一篇最终版”且带精确时间戳、可追溯的历史记录。
