name: mode-switcher

description: 把“轻松模式 / 代码模式 / 思考模式”等中文口令，映射成具体要执行的 /model 切换命令。

---

Modes

  • 轻松模式 → /model openai/gpt-5-mini
  • 代码模式 → /model openai/gpt-5.1-codex
  • 思考模式 → /model openai/gpt-5.1
  • 旗舰模式 → /model openai/gpt-5.4

Behavior

When tiger says a mode name in Chinese, do **not** silently switch models yourself. Instead:

1. Recognize the mode keyword in the user message:

- Contains "轻松模式" → light mode

- Contains "代码模式" → coding mode

- Contains "思考模式" → thinking mode

- Contains "旗舰模式" → flagship mode

2. Reply in Chinese, briefly confirming the mode, and **print the exact CLI command** tiger should run, for example:

- 思考模式: \n/model openai/gpt-5.1

- 代码模式: \n/model openai/gpt-5.1-codex

- 轻松模式: \n/model openai/gpt-5-mini

- 旗舰模式: \n/model openai/gpt-5.4

3. Do not claim that the underlying model has actually changed; only tell tiger what to execute.

Examples

  • User: "切换到思考模式"

Assistant: "要切到思考模式，请在 CLI 里执行:

/model openai/gpt-5.1"

  • User: "来点轻松模式"

Assistant: "要切到轻松模式，请执行:

/model openai/gpt-5-mini"

  • User: "用代码模式回答"

Assistant: "要用代码模式，请执行:

/model openai/gpt-5.1-codex"

  • User: "切到旗舰模式"

Assistant: "要切到旗舰模式，请执行:

/model openai/gpt-5.4"
