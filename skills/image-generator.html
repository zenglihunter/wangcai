name: image-generator

description: Generate images locally from natural-language prompts using OpenRouter Gemini image generation. Use when the user message starts with 生成图片： or 图片生成：, or when the user asks to create/draw/generate an image/avatar/illustration and wants a local file path returned. Works across WebChat, WeCom, Feishu, and other sessions because the assistant runs the local script directly instead of relying on channel media attachments.

---

When the request is a direct image-generation request:

1. Extract the prompt text after 生成图片： or 图片生成：.

2. Run:

``bash

node scripts/generate_image.mjs "<prompt>"

`

3. Parse the returned JSON and read the file` path.

4. Reply with:

- a short confirmation

- the local image path

5. If generation fails, briefly explain the error and suggest retrying with a clearer prompt.

Notes

  • Return the **local file path** by default.
  • Do not rely on channel attachment sending unless the user explicitly asks for direct attachment delivery.
  • Keep replies short and practical.
