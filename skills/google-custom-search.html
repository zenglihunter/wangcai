Google Custom Search

Use the Google Custom Search JSON API (via the provided API key) when Brave search is unavailable or when the user explicitly requests Google results. Covers credential setup, query parameter selection, the helper script for issuing requests, and how to summarize the response payload.

When to use this skill

  • Brave/other search tools are offline or blocked, and the user still needs live web results.
  • The user explicitly says “use Google Custom Search/Programmable Search Engine”.
  • We already have a concrete URL (like pricing pages) but still need broader Google results for context or corroboration.

Helper script

Use scripts/google_cse_search.py for deterministic results. It wraps the HTTP call and prints normalized JSON。默认所有查询都带上 --language lang_en --country us，保证结果以英文/美国地区为主。

# basic usage – fetch 5 results (lang_en + gl=us)
      scripts/google-custom-search/scripts/google_cse_search.py "华尔街日报 伊朗 最新" --language lang_en --country us --pretty
      
      # fetch more results (max 10)
      scripts/google-custom-search/scripts/google_cse_search.py \
        "us treasury yield" --language lang_en --country us --num 10 --pretty
      
      # override env vars ad-hoc
      scripts/google-custom-search/scripts/google_cse_search.py \
        "electric vehicle sales" --language lang_en --country us --cx <cx_id> --api-key <api_key>

Script output example:

{

      }

Use the results list to craft citations/summaries; include the link for user verification.

Manual curl (fallback)

If the script ever fails, a direct curl works the same way:

curl "https://www.googleapis.com/customsearch/v1?key=$GOOGLE_CSE_API_KEY&cx=$GOOGLE_CSE_CX&q=$(python3 -c 'import urllib.parse,sys;print(urllib.parse.quote(sys.argv[1]))' "查询词")&num=5" \
        | jq

Make sure the query is URL-encoded.

Result handling

  1. Prefer quoting the title + snippet + URL.
  2. Highlight whether the snippet is news, blog, docs, etc.
  3. Mention if total results is zero → tell the user no hits.
  4. Rate limits: default 100 queries/day; if you hit dailyLimitExceeded, explain the limit and pause.

Troubleshooting

  • keyInvalid or ipRefererBlocked: double-check the API key / allowed hosts.
  • missing or cxInvalid: confirm the correct Programmable Search Engine ID with tiger.
  • Slow responses: log searchInformation.searchTime and mention the latency.
